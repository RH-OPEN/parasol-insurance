== オリジナルアプリ

初めてアプリケーションに接続したときに表示されるのは、新しいバージョンです。

ただし、どのように変化したのかを確認するために、まず元のバージョンに移動して、それがどのようになっているかを確認してみましょう。

- 左下の **Original App** メニューをクリックします。
+
[.bordershadow]
image::05/original-app-link.png[]

- 元のアプリでは、{company-name} で使用されていた古いアプリケーションがどのようなものであるかを確認できます。各 Claim に移動（ 'Claim 1', 'Claim 2' と 'Claim 3' のタブをクリック） すると、インターフェイスが非常に粗雑で、最初に受信した電子メールのコピーと写真が添付されているだけであることがわかります。
+
[.bordershadow]
image::05/original-app-1.png[]

== プロトタイプアプリ

プロトタイプ アプリは、取締役会に公開するために構築した新しいアプリケーションの最初のバージョンです。もちろん、すべての機能がこのプロトタイプに実装されているわけではありませんが、請求処理を支援する新しいアプリケーションの可能性を示すには十分でした。

- 左側のメニューから 3 番目の項目 Claims をクリックします。
+
[.bordershadow]
image::05/claims-link.png[]

- これで、アプリケーションが受け取ったすべての Claim のリストが作成されました。各 Claim に移動（subject をクリック）すると、インターフェイスがはるかにユーザーフレンドリーになっていることがわかります。Claim をナビゲートして、Claim に添付されている画像を確認するための優れたインターフェイスが備わっています。
+
[.bordershadow]
image::05/new-app-claims.png[]

- さらに、処理された Claim については、この Claim の概要、顧客感情分析、Claim から抽出されたイベントの場所と時刻、および AI によって処理された写真を確認できます。
+
[.bordershadow]
image::05/new-app-1.png[]

- また、請求処理を支援してくれるアシスタントにもアクセスできるようになりました。開いている Claim の右下にある 'Chat icon' をクリックしてアシスタントを開きます。
+
[.bordershadow]
image::05/new-app-chat-icon.png[]

- アシスタントが開き、Claim について質問できるようになります。 RAG ノートブックで見てきたように、アシスタントはいくつかのドキュメントを頼りに質問に答えることができます。たとえば、**CLM202401** という Claim については、「ダニエルには赤信号で通過する権利がありましたか?」と試してみます。
+
[.bordershadow]
image::05/new-app-chat.png[]

これは明らかに非常に単純で粗雑なアプリケーションですが、これがプロトタイプであることを忘れないでください。そして、AI によって強化されたアプリケーションの可能性を取締役会に示し、プロジェクトへの投資を説得するには十分でした。
